    Module dentist.commands.collectPileUps.pileups

    This is the algorithm for building pile ups.

    Pipeline

    See build for the implementation.

    1. Group alignments by read ID and deduce graph edges See also: collectScaffoldJoins!collectReadAlignments(candidates)
    2. Create edges for inputGaps (makeScaffoldJoin!GapSegment)
    3. Stage `"raw"`: Build the scaffold graph from previously created edges and the "default" edges for the contigs See also: buildScaffold
    4. Stage `"resolvedBubbles"`: Detect small cycles (bubbles) and try to resolve them by identifying "skipping" edges See also: resolveBubbles, options.maxBubbleResolverIterations, options.maxBubbleSize
    5. Stage `"unambiguous"`: Resolve forks in the graph by selecting either one edge that has a significantly larger pile up than all others or remove all conflicting edges. See also: options.bestPileUpMargin, options.existingGapBonus
    6. Stage `"minSpanningEnforced"`: Removes edges with insufficient number of spanning reads See also: enforceMinSpanningReads, options.minSpanningReads
    7. Stage `"inputGapsRemoved"`: Remove input gaps from the graph as they are no longer required See also: removeInputGaps
    8. Stage `"extensionsMerged"`: Merge extending pile ups with adjacent spanning pile ups if requested See also: options.mergeExtensions, mergeExtensionsWithGaps
    9. Collect the pile ups from the scaffold graph and return them. See also: collectPileUps
