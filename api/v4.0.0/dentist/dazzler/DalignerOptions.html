    Enum DalignerOptions

    Options for daligner.

    enum DalignerOptions : string { ... }

    Enum members

    NameDescription
    asymmetric If the -A option is set (“A” for “asymmetric”) then just overlaps where the a-read is in block X and the b-read is in block Y are reported, and if X = Y then it further reports only those overlaps where the a-read index is less than the b-read index.
    averageCorrelationRate Searching for local alignments involving at least -l base pairs (default 1000) or more, that have an average correlation rate of -e (default 70%).
    bandWidth Search code looks for a pair of diagonal bands of width 2^^w (default 26 = 64) that contain a collection of exact matching k-mers (default 14) between the two reads, such that the total number of bases covered by the k-mer hits is h (default 35).
    bridge Bridge consecutive aligned segments into one if possible
    hitBaseCoverage Search code looks for a pair of diagonal bands of width 2^^w (default 26 = 64) that contain a collection of exact matching k-mers (default 14) between the two reads, such that the total number of bases covered by the k-mer hits is h (default 35).
    identity If the -I option is set (“I” for “identity”) then when X = Y, overlaps between different portions of the same read will also be found and reported.
    kMerSize Search code looks for a pair of diagonal bands of width 2^^w (default 26 = 64) that contain a collection of exact matching k-mers (default 14) between the two reads, such that the total number of bases covered by the k-mer hits is h (default 35).
    masks If there are one or more interval tracks specified with the -m option (m for mask), then the reads of the DB or DB’s to which the track applies are soft masked with the union of the intervals of all the interval tracks that apply, that is any k-mers that contain any bases in any of the masked intervals are ignored for the purposes of seeding a match.
    maxKmerMemory Let the program automatically select a value of t that meets a given memory usage limit specified (in Gb) by the -M parameter.
    maxKmerOccurence Suppresses the use of any k-mer that occurs more than t times in either the subject or target block.
    minAlignmentLength Searching for local alignments involving at least -l base pairs (default 1000) or more, that have an average correlation rate of -e (default 70%).
    minAReadLength By setting the -H parameter to say N, one alters daligner so that it only reports overlaps where the a-read is over N base-pairs long.
    modimerPercentage Modimer percentage (take % of the k-mers)
    numThreads The program runs with 4 threads by default, but this may be set to any power of 2 with the -T option.
    sortMap sort .las by A-read,A-position pairs for map usecase; off => sort .las by A,B-read pairs for overlap piles
    strongCompositionalBias If the -b option is set, then the daligner assumes the data has a strong compositional bias (e.g. >65% AT rich).
    tempDir
    tracePointDistance The local alignments found will be output in a sparse encoding where a trace point on the alignment is recorded every -s base pairs of the a-read (default 100bp).
    verbose
